# مدیریت Pull Requestهای Merge شده بدون تایید

اگر یک Pull Request به صورت ناخواسته در حالت `Merged` یا `Closed` قرار گرفته و گزینهٔ **Approve** در دسترس نیست، مراحل زیر را دنبال کنید تا بتوانید تغییرات را با تایید لازم دوباره ادغام کنید.

## بازگردانی شاخه و ساخت PR جدید
1. اگر دکمهٔ **Restore branch** در صفحهٔ PR فعال است، آن را بزنید تا شاخهٔ مربوطه دوباره ایجاد شود.
2. پس از بازگردانی شاخه، از آن برای ساخت یک Pull Request جدید استفاده کنید.

## دریافت تایید از یک Reviewer دیگر
1. در PR جدید، از بخش **Reviewers** یک بازبین (شخصی غیر از نویسندهٔ PR) اضافه کنید.
2. بازبین باید از تب **Files changed** روی **Review changes** کلیک کرده و گزینهٔ **Approve** را انتخاب کند.
3. بعد از ثبت تایید، PR را Merge کنید.

## جلوگیری از تکرار مشکل با قوانین Branch Protection
برای شاخهٔ `main` به مسیر **Settings → Branches** بروید و یک قانون Branch Protection با تنظیمات زیر بسازید:
- **Require a pull request before merging** را فعال کنید.
- مقدار **Require approvals** را روی ۱ (یا عددی بالاتر متناسب با فرآیند تیم) تنظیم کنید.
- (اختیاری) گزینه‌های **Require review from Code Owners** و **Dismiss stale approvals** را فعال نمایید تا کنترل بیشتری روی تاییدها داشته باشید.

با انجام این مراحل، اطمینان حاصل می‌کنید که هر PR قبل از Merge شدن حداقل یک تایید معتبر دریافت می‌کند و از تکرار ادغام‌های بدون تایید جلوگیری می‌شود.
